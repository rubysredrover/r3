02

The Problem

Emotion AI fails for people with CP.

What generic AI sees

Grimacing = angry
Slow movement = sad
Muscle tension = stressed
Stimming = agitated

What MARS sees

Grimacing = spasticity — check the eyes
Slow movement = motor fatigue, not mood
Muscle tension = she’s actually happy
Stimming = excitement — this is joy

What We Built in 48 Hours

CP-aware emotion intelligence
for the MARS robot.

01

Emotion Tracker

Two-tier detection: InspireFace on-device (50ms, free) + Gemini Vision (CP-aware prompts, on mood change only). Runs on Jetson Orin Nano.

02

Ruby Score

Eye contact (45%) + volume (25%) + response speed (30%) = 0-100 engagement score. Calibrated to Ruby’s baseline, not population averages.

03

Trust Layer

Mom controls who sees Ruby’s data. Bolo grants with instant revocation. PCA gets mood:read. Grandma gets mood:read. Random person gets denied.

Architecture

Two tiers. Sovereign by default.

👀

On-Device (Fast Path)

InspireFace on Jetson. Face ID + basic emotion. Zero API calls. 50ms. Free.

↓ mood changed?
🤖

Gemini Vision (Deep Path)

CP-aware prompts. Distinguishes spasticity from anger. Only fires on mood change or distress.

📈

Ruby Score + Alert

Eye + volume + speed = 0-100. Logged. Trended. Mom gets alerted if it drops below 20.

Adaptive timing: 5s during distress, 30s when stable. Cuts API calls by ~80%.
